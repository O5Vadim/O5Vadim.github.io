HTML — Структура страницы

HTML это не язык программирования — это разметка. Как Word только в коде.

Базовая структура

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Название страницы</title>
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <!-- Тут весь контент -->
</body>
</html>

Ссылки

Обычная ссылка

<a href="https://google.com">Открыть Google</a>

Ссылки между страницами сайта

<!-- С главной на wiki/ -->
<a href="wiki/">Перейти на Вики</a>

<!-- Из wiki/ на главную -->
<a href="/">На главную</a>

<!-- Из wiki/ в http/ -->
<a href="../http/">Документация</a>
💡 ../ означает "выйди из папки на уровень выше"

Открыть в новой вкладке

<a href="https://google.com" target="_blank">Google</a>

Кнопка как ссылка

<a href="/wiki/">
    <button class="кнопка">Перейти на Вики</button>
</a>
Картинки

Добавить картинку

<img src="фото.jpg" alt="Описание фото">

Из интернета

<img src="https://сайт.com/картинка.jpg" alt="Описание">

Пути к картинкам

<!-- Рядом с index.html -->
<img src="cat.jpg" alt="Кот">

<!-- В папке images/ -->
<img src="images/cat.jpg" alt="Кот">

<!-- Уровнем выше -->
<img src="../images/cat.jpg" alt="Кот">

Размер

<img src="cat.jpg" alt="Кот" width="300">

<!-- Или через CSS -->
<img src="cat.jpg" alt="Кот" style="width:100%; max-width:500px">
⚠️ Всегда заполняй alt — это описание для поисковиков
Структура папок сайта

Папка = подссылка. Файл внутри всегда называется index.html

мой-сайт/
├── index.html          → сайт.com/
├── style.css           → стили
├── sw.js               → Service Worker
├── wiki/
│   └── index.html      → сайт.com/wiki/
├── http/
│   └── index.html      → сайт.com/http/
└── images/
    └── cat.jpg

Подключить style.css из подпапки

<!-- Из wiki/index.html -->
<link rel="stylesheet" href="../style.css">

HTTP коды ответа
💡 Главное правило: 4xx — виноват клиент, 5xx — виноват сервер

1xx — Информационные

  • 100
    Continue
    Продолжай отправку данных
  • 101
    Switching Protocols
    Переключение протокола, например на WebSocket

2xx — Успех ✅

  • 200
    OK
    Всё хорошо
  • 201
    Created
    Ресурс создан
  • 204
    No Content
    Запрос принят, ответа нет

3xx — Перенаправления 🔀

  • 301
    Moved Permanently
    Переехал навсегда. Используется для http → https
  • 302
    Found
    Переехал временно
  • 304
    Not Modified
    Страница не изменилась — бери из кэша

4xx — Ошибки клиента 🙋

  • 400
    Bad Request
    Кривой запрос
  • 401
    Unauthorized
    Нужно войти в аккаунт
  • 403
    Forbidden
    Вход запрещён даже с аккаунтом
  • 404
    Not Found популярный
    Страница не найдена. Интернет ЕСТЬ — сервер просто не нашёл страницу!
  • 408
    Request Timeout
    Сервер устал ждать и закрыл соединение
  • 410
    Gone
    Страница была но удалена навсегда
  • 413
    Payload Too Large
    Файл слишком большой
  • 418
    I'm a Teapot пасхалка
    Я чайник! Первоапрельская шутка 1998 года. Попробуй: google.com/teapot
  • 429
    Too Many Requests
    Слишком много запросов — притормози

5xx — Ошибки сервера 💥

  • 500
    Internal Server Error
    Что-то сломалось на сервере — не твоя вина!
  • 502
    Bad Gateway
    Промежуточный сервер получил плохой ответ
  • 503
    Service Unavailable
    Сервер перегружен или на обслуживании
  • 504
    Gateway Timeout
    Промежуточный сервер не дождался ответа
Service Worker — Офлайн режим

Подключить в каждом HTML файле

<script>
    if ('serviceWorker' in navigator) {
        navigator.serviceWorker.register('/sw.js')
    }
</script>

Файл sw.js в корне сайта

// Меняй версию каждый раз когда обновляешь сайт!
const CACHE = 'интернет-офлайн-v2'

const FILES = [
    '/',
    '/index.html',
    '/wiki/',
    '/wiki/index.html',
    '/http/',
    '/http/index.html',
    '/style.css'
]

self.addEventListener('install', e => {
    e.waitUntil(
        caches.open(CACHE).then(cache => cache.addAll(FILES))
    )
})

self.addEventListener('activate', e => {
    e.waitUntil(
        caches.keys().then(keys =>
            Promise.all(keys.filter(k => k !== CACHE).map(k => caches.delete(k)))
        )
    )
})

self.addEventListener('fetch', e => {
    e.respondWith(
        fetch(e.request).catch(() => caches.match(e.request))
    )
})
⚠️ Когда добавляешь новую страницу — добавь её в FILES и смени версию v2 → v3!
GitHub Pages

Создать сайт

1. Репозиторий называется: твойник.github.io
2. Загрузи файлы через Add file → Upload files
3. Settings → Pages → Branch: main → Save
4. Через пару минут: твойник.github.io ✅

Создать папку на GitHub

Add file → Create new file
Имя файла: wiki/index.html
GitHub сам создаст папку!
